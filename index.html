    Linux下不重启永久修改hostname 

    hostname 主机名

    这条命令，可以修改hostname立即生效。但是重启后，就会失效。
    要想重启后生效，修改vi /etc/sysconfig/network文件中的HOSTNAME项，如果没有该项，则新增即可。 

    [root@T-server ~]# cat /etc/sysconfig/network
# Created by anaconda
NETWORKING_IPV6=no
PEERNTP=no
HOSTNAME=T-server

    测试
    使用hostname查看即可：

    [root@T-server ~]# hostname
T-server

  2. 使用Supervisor管理linux服务

    作者 Bug管理员
    分类 架构 | 标签: 架构 linux
    发布于 2019-07-07 21时50分10秒 星期日

    为什么使用supervisor

    服务器上往往会启动着许多的服务，跑着许多脚本。随着服务器上服务的增多，管理起来就变的繁琐，甚至的当有的脚本挂了，我们可能都不能一眼看出少了什么服务。
    向大家推荐 Supervisor。
    Supervisor可以对进程进行管理，当管理的进程在崩溃时，会自动重启该进程。最早也是因为这个功能，接触的Supervisor。当时项目中有个服务时不时的会挂掉，一时间也没有找到原因，后在朋友推荐下，用上了Supervisor，然后慢慢的分析挂掉的原因。

    有的服务是由一组自服务组成的，它们重启的时候也比较麻烦，万一手抖还可能弄错。supervisor支持对进程组的管理，只需要简单的配置就可以实现。

    安装

    安装，以cenOS7为例：

    pip install supervisor

    作者 Bug管理员
    分类 数据库 | 标签: 数据库 MySQL

    分区：（应用层感知不到变化）

    将把一张表的数据根据分区逻辑均衡分摊到多个存储介质中。在逻辑上看还是只有一张表，但实际被分成了N个物理块。分区类型有（range、list、hash、key）。

    主要解决单表数据量过大，读写数据、修改数据时重建索引效率等问题。

    一般当一张表的查询，在优化后，查询速度仍比较慢时，就需要选择合适的业务字段进行分区。

    相对分表，几乎无工作量。应用端一般无需修改代码。

    分表：mycat、tidb

    把一张表按一定规则，拆分成N张表的过程，主要是为了解决单表数据量过大时，单表压力过大的问题。分横向拆分和纵向拆分。

    横向拆分：

    将一张表中的不同数据字段，拆分到2张或者多张表中。一般横向拆分的情况有：表字段过多，字段内容过大（如text，blob等），不经常使用等。
