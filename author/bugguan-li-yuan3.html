    Hashtable简介

    Hashtable和HashMap类似，同样是基于哈希表实现的。Hashtable是线程安全的（有许多方法用synchronized修饰），key和value都不支持null，没有红黑树的实现。且已不建议使用。

    与HashMap比较

    Hashtable与HashMap类似都实现了Map、Cloneable, Serializable接口:
    Hashtable继承自Dictionary类，HashMap继承自AbstractMap类。
    Hashtable不允许记录的键或者值为null，HashMap允许记录的键和值为null。
    Hashtable初始容量为11，HashMap的初始1容量为16。 Hashtable的许多方法中使用了synchronized关键字，所以Hashtable是线程安全的，即任一时刻只有一个线程能写Hashtable，这也导致了Hashtable在写入时会比较慢。 如果在非线程安全的情况下，建议使用HashMap。如果在线程安全的情况下可以使用Hashtable，不过官方还是建议使用ConcurrentHashMap。

    构造函数

    大致上和HashMap没有太大区别，主要的有以下两个区别：
    1、无参构造函数，初始的~initialCapacity~是11

    this(11, 0.75f);
}

  4. Java Map

    by Bug管理员
    in java on 2018-03-26 | tags: java 集合

    前言

    Map是java.util中的集合类，另一个和它一样常用的是List。
    Map提供了一个方便的数据结构，用来存储键值对。
    Map根据键得到值，因此不允许键重复(重复就覆盖了)，但允许值重复。
    它有四个实现类，分别是HashMap、Hashtable、LinkedHashMap和TreeMap。

    Hash算法简述

    常见的散列函数方法：除留余数法、直接寻址法、数字分析法、平方取中法、折叠法、随机数法
    常见的处理Hash冲突的方法：拉链法、线性探索法等。

    Hashmap

    HashMap是最常用的Map，它根据键的HashCode值存储数据，根据键可以直接获取它的值，具有很快的访问速度，遍历时，取得数据的顺序是完全随机的。
    HashMap最多只允许一条记录的键为null;
    允许多条记录的值为null;
    HashMap不支持线程的同步，即任一时刻可以有多个线程同时写HashMap，可能会导致数据的不一致。 如果需要同步，可以用 Collections的synchronizedMap方法使HashMap具有同步的能力，或者使用ConcurrentHashMap。

  5. HTTP请求中GET和POST的区别

    by Bug管理员
    in other on 2017-05-28 | tags: http java

    1、很显然，报文方法名不同，一个是GET，一个是POST。

    2、请求携带参数的方式不同
    GET参数常规的是跟在请求的url后面，以?开始，参数间以&隔开，数据被直观的展现在URL上。如：https://www.baidu.com/s?ie=UTF-8&wd=test
    POST参数一般是放在请求体entity-body中，传输给目标服务器，数据在URL中不可见。

    3、请求时，携带的参数大小限制不同
    GET请求的URL传参有大小限制
    POST请求理论上是没有大小限制的，但实际情况是，各类应用服务器都对请求大小做了限制。因为过大请求包，会增加服务器解析时间。甚至会被人恶意请求超大的数据包。
