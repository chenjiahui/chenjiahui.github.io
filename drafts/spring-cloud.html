spring cloud前言

作者 Bug管理员
分类 Java | 标签: java spring-cloud 架构
发布于 2018-07-15 10时38分52秒 星期日

从2011年开始接触spring，记得当时使用的是3.x版本，到后来使用4.x版本。到后来使用到maven，spring boot，spring cloud。 Java语言一直在高速发展，尤其是spring的生态也越发的完善、好用、简便。

说到spring，很多刚入行的可能会被Spring、SpringMVC、SpringBoot、SpringCloud搞的摸不着头脑。
那么他们之间是怎么样的关系呢？

Spring
Spring Framework是一个开源的大家族，其核心是一个轻量级的控制反转（IoC）和面向切面（AOP）的容器,Java 开发框架，至于控制反转、面向切面、轻量级、容器等。
spring的家庭成员有很多，如，Spring AOP、Spring ORM、spring JDBC、spring MVC、spring Security、Spring JMS、Spring JPA等。

Spring MVC
spring MVC是spring生态的一部分，MVC是一种优秀web项目开发模式，MVC其实取自V（View：视图、用户客户端）、M（Moudle：模块，业务层）和C（Control控制层）。spring MVC就是spring版的MVC模式的实现。

Spring Boot
使用过spring、spring MVC的人，因为spring框架的灵活性和可扩展性，spring提供了很多配置项，随着项目的发展，spring配置文件也越来越繁琐（虽然后期用很多注解，减少了不少xml配置代码）。 而这些年大火的python、nodejs等语言的web框架，就配置非常简单。所以spring社区推出了Spring Boot，它的目的在于实现自动配置，降低项目搭建的复杂度。

Spring Cloud
随着互联网的崛起，互联网架构也发生着变化，微服务架构得到广泛的应用。 得益于Spring Boot的配置简化，Spring Cloud利用其服务的自动化注册与发现的理念进行治理，将一系列的独立的微服务整合到了一起，并实现了动态增加服务。 不仅仅是Java服务，只要实现了Spring Cloud的服务注册协议。其他语言的服务也可以注册进去。

Spring Cloud的服务众多，常用的有
服务发现——Eureka 传送门
客服端负载均衡——Ribbon
断路器——Hystrix
服务网关——Zuul
分布式配置——Spring Cloud Config
