为什么使用conda

在项目开发过程中，会涉及到团队间的协作，而团队间的代码环境也会存在这一些差异。 也有些特殊的模块，必须使用python2或者python3，我们知道，py2和py3是不太兼容的。 这就使我们不得不去实现同一服务器的多python环境。 至于为什么不使用docker，因为docker还不太熟悉，docker而且不太稳定，不确定的因为比较多。

安装conda

conda创建环境

比如我的mac使用的是python2.7的环境，我现在创建一个python3.6的环境，并命名为py3.6 conda create -n py3.6 python=3.6 一顿安装之后，会有如下提示，就说明安装成功了，并且还说明了如何激活环境：

#
# To activate this environment, use
#
#     $ conda activate py3.6
#
# To deactivate an active environment, use
#
#     $ conda deactivate
