centOS上svn的安装与迁移

作者 Bug管理员
分类 运维 | 标签: svn linux centOS
发布于 2017-07-19 10时23分46秒 星期三

系统环境

系统环境：centos7.2

安装

yum -y install subversion

创建版本库目录

mkdir -p /data/cjh/svnrepos/

使用svnadmin创建版本库

svnadmin create /data/cjh/svnrepos/

版本库中文件列表

ls /data/cjh/svnrepos/
drwxr-xr-x 2 root root 4096 Mar 27 10:16 conf
drwxr-sr-x 6 root root 4096 Jul 19 23:08 db
-r--r--r-- 1 root root    2 Jun 27  2017 format
drwxr-xr-x 2 root root 4096 Jun 27  2017 hooks
drwxr-xr-x 2 root root 4096 Jun 27  2017 locks
-rw-r--r-- 1 root root  229 Jun 27  2017 README.txt

配置conf

conf目录下一共3个文件
passwd配置svn用户账号密码的
authz配置各个账号所拥有的目录权限。可以对用户分组，设置读写权限等。
svnserve.conf是svn服务器的配置文件

具体的各个配置项有什么作用，各个配置文件中都有介绍。重点说下几个主要配置。

svn服务配置

anon-access = none：表示禁止匿名用户访问。设置read表示允许匿名用户只读。
auth-access = write：表示授权用户拥有读写权限。
password-db = passswd：指定用户名口令文件，即 passwd 文件。
authz-db = authz：指定权限配置文件，即 authz 文件。

用户账号密码配置

vi passwd，在[users]下设置用户名和密码行一个用户设置一个账号为cjh密码为cjh123456的账户 

[users]
cjh=cjh123456

账号权限配置

账号分组

在[groups]标签下设置分组，多个分组以换行分隔。

[groups]
#如设置`cjh,cjh2`2个账号为`admin`分组。
admin=cjh,cjh2,cjh3
#设置`ios1,ios2,ios3`账号为iOS分组。
ios=ios1,ios2,ios3

目录权限

#如根目录只允许admin分组的用户访问
[/]
@admin=rw
#设置分组只是为了方便管理用户，也可以不设置分组，直接标明账号权限。
#如下，设置ios组用户对ios项目的读写权限，设置白盒测试人员test1账号，对ios项目的读权限
[/iosproject]
@ios=rw
test1=r

[/androidproject]
android1=rw
android2=rw
test2=r

启动svn

svnserve -d -r /data/cjh/svnrepos/

svn项目的迁移

dump

如果是新老服务器在内网网络是打通了的情况下，建议直接使用dump方式，一般dump文件都比较大 到原svn服务器下生成dump文件。 svnadmin dump /data/svn/>/data/2017-07-19-svn.dump

将dump文件通过scp命令发送文件到新服务器上

scp ./data/2017-07-19-svn.dump 192.168.1.170:/data

在新服务器上将dump文件读进SVN仓库

svnadmin load /data/cjh/svnrepos/<2017-07-19-svn.dump

启动svn

svnserve -d -r /data/cjh/svnrepos/

测试后，记得及时通知同事切换svn，并关闭老的svn服务，防止错误提交。

ps: 推荐给svn地址配置上域名解析，那样下次迁移svn时，SVN的用户们就不需要去切换操作了。
