    作者 Bug管理员

    发布于 2017-07-08 20时54分19秒 星期六

    最近有同事跟我说，有个Java项目的内存使用率有点高。
    那个项目并没有承载什么大的业务，按理不应该占用太多的内存，很有可能是发生了内存泄漏。
    那么内存泄漏，该怎么解决，如何才能找到内存泄漏的代码块呢？
    这里推荐一款Java堆内存分析工具MAT(Memory Aanlysis Tool)

    安装

    可以在Eclipse插件市场 help->Eclipse Marketplace，查找插件Memory Aanlysis
    如果不想Eclipse中装太多插件，也可以去MAT download页面下载最新版的MAT安装程序。

    使用

    打开MAT，可以在File-> Open Heap Dump中导入待分析的HeapDump文件。

    如何获得堆转储文件

    可以使用jmap -dump获取：

    jmap -dump:format=b,file=<dumpfile.hprof> <pid>

  3. HTTP请求中GET和POST的区别

    作者 Bug管理员
    分类 other | 标签: http java
    发布于 2017-05-28 21时27分07秒 星期日

    1、很显然，报文方法名不同，一个是GET，一个是POST。

    2、请求携带参数的方式不同
    GET参数常规的是跟在请求的url后面，以?开始，参数间以&隔开，数据被直观的展现在URL上。如：https://www.baidu.com/s?ie=UTF-8&wd=test
    POST参数一般是放在请求体entity-body中，传输给目标服务器，数据在URL中不可见。

    3、请求时，携带的参数大小限制不同
    GET请求的URL传参有大小限制
    POST请求理论上是没有大小限制的，但实际情况是，各类应用服务器都对请求大小做了限制。因为过大请求包，会增加服务器解析时间。甚至会被人恶意请求超大的数据包。

  4. java虚拟机-垃圾回收GC

    作者 Bug管理员
    分类 java | 标签: java jvm
    发布于 2017-04-30 20时57分50秒 星期日

    垃圾回收算法

    ￼

    垃圾检测

    引用计数法
    给对象添加一个引用计数器，被引用时+1，失效时-1。垃圾回收时，计数为0的对象。此算法最致命的是无法处理循环引用的问题。
    可达性检测
    以根对象为起点，进行搜索，如果对象不可达，则认为该对象为垃圾对象。

    那根节点是怎么确定的呢(GC Root)
    虚拟机栈（栈帧中的本地变量表）中引用的对象
    方法区中类静态属性引用的对象
    方法区中常量引用的对象
    本地方法栈中JNI（即一般说的Native方法）引用的对象
    这些信息都使用一组称为OopMap的数据结构来实现

    引用类型

    强引用
    只要强引用还存在，垃圾收集器永远不会回收掉被引用的对象

    软引用
    描述一些还有用但并非必需的对象。在系统将要发生内存溢出异常之前，将会把这些对象列进回收范围之中进行第二次回收。如果这次回收还没有足够的内存才会抛出内存溢出异常

    弱引用
    用来描述非必需的对象，强度比软引用还要弱，被弱引用关联的对象只能生存到下一次垃圾收集发生之前，当垃圾收集器工作时，无论当前内存是否足够，都会回收掉只被弱引用关联的对象

    虚引用
    也称为幽灵引用或者幻影引用，是最弱的一种引用关系。一个对象是否有虚引用的存在，完全不会影响其生存时间，也无法通过虚引用来取得一个对象实例。为一个对象设置虚引用关联的唯一目的就是能在这个对象被收集器回收时收到一个系统通知 …

    Java内存模型

    内存区包含方法区、虚拟机栈、本地方法栈、堆、程序计数器。如下图：

    方法区
    方法区属于线程共享的内存区域，存放已经被虚拟机加载的类的结构信息、常量池、静态变量、构造函数等信息。
    Java堆
    Java Heap属于线程共享的内存区域，存储java实例的地方。是GC管理的主要区域。
    虚拟机栈
    java栈总是和线程关联再一起的，每创建一个线程，jvm就会为线程创建一个栈桢，用于存储局部变量、方法返回值、动态链接方法、操作数栈等。
    本地方法栈
    本地方法栈属于线程私有的数据区域，这部分主要与虚拟机用到的 Native 方法相关，一般情况下，我们无需关心此区域。
    程序计数器
    它是内存中的一小块空间，是用来标示下一条字节码指令的行号的计数器。
    字节码解释器在工作时，如果线程执行的是Java方法，则程序计数器的记录就是下一条执行的指令的地址。如果线程执行的是本地方法，则程序计数器值为undefined 如分支、循环、跳转、异常处理、线程恢复等都用到了方法区。
