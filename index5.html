    前言

    Map是java.util中的集合类，另一个和它一样常用的是List。
    Map提供了一个方便的数据结构，用来存储键值对。
    Map根据键得到值，因此不允许键重复(重复就覆盖了)，但允许值重复。
    它有四个实现类，分别是HashMap、Hashtable、LinkedHashMap和TreeMap。

    Hash算法简述

    常见的散列函数方法：除留余数法、直接寻址法、数字分析法、平方取中法、折叠法、随机数法
    常见的处理Hash冲突的方法：拉链法、线性探索法等。

    Hashmap

    HashMap是最常用的Map，它根据键的HashCode值存储数据，根据键可以直接获取它的值，具有很快的访问速度，遍历时，取得数据的顺序是完全随机的。
    HashMap最多只允许一条记录的键为null;
    允许多条记录的值为null;
    HashMap不支持线程的同步，即任一时刻可以有多个线程同时写HashMap，可能会导致数据的不一致。 如果需要同步，可以用 Collections的synchronizedMap方法使HashMap具有同步的能力，或者使用ConcurrentHashMap。

  5. python图像处理库PIL读取后，保存图片变模糊

    作者 Bug管理员
    分类 python | 标签: python pil
    发布于 2017-11-28 21时19分21秒 星期二

    最近在给活动图片批量加入二维码的时候，发现图片在使用PIL的paste()方法在指定区域粘贴二维码，保存之后，非处理区域的图像也变模糊了。

    检查发现，在原先空白处增加二维码之后，图片大小本应该比原图增加的，而结果却是比原图更小。 显然图片在save的时候，使用了压缩算法对图片进行了压缩处理。

    查阅资料及源码，发现，图片在保存的时候，可以使用quality参数指定保存图像的质量。

    quality参数的取值范围从1（最差）到95（最佳）。 默认值为75，使用中应尽量避免高于95的值;

    为什么最佳取值不是100呢？因为100会禁用部分JPEG压缩算法，并导致大文件图像质量几乎没有任何增益。

    因为代码中未设置quality参数，所以程序使用了默认值75，导致图片变模糊了。

    img(img_name, quality=95)
